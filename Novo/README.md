# Novo – Repositório de Conhecimento para Vibecoding

Esta pasta será usada como **cérebro** do projeto, armazenando informações de sistema, dependências, variáveis e outras referências que podem ser consumidas por IAs para melhorar o fluxo de desenvolvimento.

## Estrutura sugerida

- `config/` – arquivos de configuração e exemplos de `.env`
- `dependencies/` – listagens e diagramas das dependências do projeto
- `variables/` – explicação das variáveis de ambiente e seus propósitos
- `knowledge/` – notas, diagramas e documentos de referência
- `scripts/` – scripts auxiliares para gerar ou transformar dados

## Como usar
1. Adicione arquivos relevantes nas subpastas.
2. Documente cada item de forma clara, com exemplos de uso.
3. As IAs podem ler ou executar esses recursos para fornecer sugestões e automações mais precisas.
