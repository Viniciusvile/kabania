# Ecossistema de Dados e Cérebro para Vibecoding

Esta pasta serve como um repositório centralizado de **informações do sistema**, **dependências**, **variáveis de configuração** e **conhecimentos** que podem ser consumidos por mim ou por outra inteligência artificial para melhorar o *vibecoding*.

## Estrutura sugerida

- `config/` – arquivos de configuração (ex.: `.env.example`, `config.yaml`)
- `dependencies/` – listagens de dependências do projeto (ex.: `package.json`, `requirements.txt`, diagramas de arquitetura)
- `variables/` – descrições de variáveis de ambiente e seus propósitos
- `knowledge/` – notas, diagramas e documentos de referência sobre o domínio do problema
- `scripts/` – scripts auxiliares que geram ou manipulam dados para o IA

## Como usar

1. **Adicionar** novos arquivos relevantes a estas subpastas.
2. **Documentar** cada item com clareza, incluindo exemplos de uso.
3. **Consultar** via execução de scripts ou leitura direta pelos modelos de IA que precisam de contexto.

Esta estrutura facilita a agregação de conhecimento e permite que assistentes de IA acessem rapidamente informações críticas do sistema.
